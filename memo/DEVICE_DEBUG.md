# Device Debug

## デバイス周り

一番死ぬ確率が高い

### デバッグ順

基本的に守ること。テキトーにやると時間の無駄。

1. 物理（ケーブル / 電源）
2. 列挙（lsusb）
3. カーネルログ（dmesg）
4. デバイスノード（/dev）
5. ドライバ（lsusb -t）
6. 権限 / udev
7. プロセス競合（lsof / fuser）

### 列挙

`lsusb`

基本中の基本。USB busにデバイスが乗ってさえいれば出力に出る。ここでデバイスが出ない場合、物理/電源/デバイス自体が死亡。
それより上の層は一旦疑わない。ドライバも関係ない。`lsusb`で出るのに`/dev/xxx`が出ないならドライバをすぐ疑う。

### カーネルログ見る

`dmesg -w`

動的にカーネルリングバッファのログを見れる。USBを抜き差しして無反応なら死。ただ死んでるのがわかるというだけでなく、エラーによってどこで問題が起きているのか特定/推定することができる。`lsusb`して出てこないなら一旦叩く。過去のログなら`dmesg`だけでいい。`tail`したり`grep`すると良い。

`journalctl -kb`

`dmesg`と同じような用途で使う。何時のboot時かなどログを細かく指定したりもできる。かなり過去のログでも出せたりするが、`dmesg`ほどの手軽さはない。慣れてないなら微妙。

### デバイスノード

`ls /dev/ttyUSB*`, `ls /dev/video*`, etc.

`lsusb`で出るのにこいつが出ないなら、ドライバを疑う。間違ってもこのPathをハードコードして制御をするなど恐ろしいことをしないこと。`udev`のSYMLINKが壊れているかつ時間がない時などは仕方ないのでconfigに直で書いて動かす。ここに出るのに使えない場合はプロセス競合や権限を疑う。

### ドライババインド

`lsusb -t`

どのドライバに紐づいているか確認できる。

### udev

`udevadm info -a -n /dev/xxx`

VID/PID/iSerial/KERNEL等取る。vid/pidだけなら`lsusb`でもすぐ出せる。`lsusb -v`で`grep -A`くらいでフィルタしてやるとすぐ確認できるかもしれない。自分なりの確認方法は欲しい。`udev`書ける時間があるなら良いが、時間がないのにのこのこ.rulesを書いていてもしかたがないのでpath直書きでも絶対動かす。

`udevadm monitor`

udevの動きを動的に見れる的なやつ。SYMLINKが出ていなかったりudevが怪しいならこれ。確認したいデバイスを抜き差しして出力を見ること。

`less /etc/udev/rules.d/xxx.rules`

rulesはちゃんと`cp`した？おかしいと思ったら実際に/etcにあるudevruleを自分の目で確かめにいくこと。

### 捕まれている？

`lsof /dev/xxx`

他のプロセスにデバイスを握られているかもしれないなら即座に打つ。PID(process id)が分かれば`kill -9`でSIGKILL飛ばして殺す。dynanmixel_wizardで接続しっ放しとか注意。yanoは良く使う。ポート確認にも使える(ss的な)。

`fuser /dev/xxx`

`lsof`とは違って調査用というよりは殺し屋。`fuser -k /dev/xxx`で即殺害。

### 権限大丈夫？

`chmod 777 /dev/xxx`

`udev`でMODE設定しておけば普通は問題ない筈だが、意外と見落としがちなのがこれ。ごちゃごちゃ考えるよりも脳死で`777`。デバイスファイルだと実行権限は関係ないが、疑って早く打てる方が実用的。

`ll /dev/xxx`

念入りに確認しても良いが、udevで.rulesを書いている場合などは反映されないのでこれを安易に信じないこと。もたもたしているくらいなら、何も考えずに怪しそうなデバイスファイルにフル権限与える。


### 物理...?

コマンド打ってもよくわからないなら、取りあえずもう一回抜き差ししてみる。ポートを変えてみる(KERNELでudev縛ってるやつは注意)こと、ケーブルを変えてみること。

### カメラ

`v4l2-ctl --list-devices`

基本的に使っているのが普通のwebcam的なやつならこれででてくるはず。`/dev/video*`だとどのカメラか非常に分かり辛いので、カメラ周りのデバッグにはなるべくこっちを使う。

`v4l2-ctl -d /dev/video* --list-formats-ext`

対応フォーマット確認。MJPG本当に対応してるか？YUYVだけじゃないか？resolution(解像度)はconfigに書いたものが存在しているか？fpsは？全部ここで確認する。ただ、絶対では無いので過信はしない。

`v4l2-ctl -d /dev/video* --stream-mmap --stream-count=100`

カメラ単体でテスト。ここで死ぬのにros2に責任を擦りつけるな。
