# RoboCup Rescue Robot League 2026 詳細ルール＆仕様書まとめ

## 1. ロボットハードウェア・オペレーター要件

### 1-1. ロボットの仕様規定
* [cite_start]**重量制限:** 最大80kgまで [cite: 99]。
* [cite_start]**小型ロボット枠の基準:** 30cm四方の開口部を通過できる機体であること [cite: 100]。 // これは不可能
* [cite_start]**必須安全装備:** 機体を安全に運搬するための「安全な持ち手（セキュアハンドル）」と、ハードウェア的な「非常停止スイッチ（e-stop）」の装備が義務付けられる [cite: 101] // Hardware EMSは搭載済み
* **有線（テザー）の取り扱い:**
  * [cite_start]電源供給や通信用のテザーは常に許可される [cite: 107]。
  * [cite_start]障害物への引っ掛かりを防ぐため、ロボット側にウィンチ機構を持たせて巻き取る方式を推奨 [cite: 108]。
  * [cite_start]コース内でケーブルを引きずる場合、レーン入口に「ケーブルハンドラー（1名のみ）」を配置し管理すること [cite: 109, 110]。
  * [cite_start]ケーブルをコースの壁越しに誘導したり、棒などの道具を使ってさばくことは禁止 [cite: 110, 111]。

### 1-2. オペレーター（操縦者）の制限
* [cite_start]**ブラインド操作:** オペレーターはテストレーン内のロボットを直接目視してはならない [cite: 102]。
* [cite_start]**情報源:** 状況把握はすべて、システムインターフェース上の情報のみに依存すること [cite: 104]。 // 基本的にFoxgloveの情報を便りにする
* [cite_start]**人員:** 操縦できるのは常に1名のみ（途中の交代は可） [cite: 105][cite_start]。操縦者はステーションから離れてはならない [cite: 106]。
* [cite_start]**会話の禁止:** 操縦中、チームメンバーとの会話や合図によるコミュニケーションは禁止（リセット宣言時のみ例外） [cite: 103]。

### 1-3. 競技時間とペナルティ
* **時間配分（1枠30分）:**
  * [cite_start]5分: セットアップ [cite: 117]
  * [cite_start]20分: ロボット操作（競技時間） [cite: 118]
  * [cite_start]2分: センサー評価（被災者クレート） [cite: 119]
  * [cite_start]3分: 撤収 [cite: 119]
* [cite_start]**構成変更の禁止:** 20分の操作時間中に、ロボットの構成を変える（例：アームを取り外すなど）ことは禁止 [cite: 165][cite_start]。トライアル間であれば変更可能だが、移動機構（ロコモーション）は同一であること [cite: 166, 167]。
* **リセットのペナルティ:**
  * [cite_start]競技中にチームメンバーがロボットに触れると自動的にリセット扱いとなる [cite: 164]。
  * [cite_start]リセット時は最低2分間のペナルティタイムが経過するまで再開できず、ロボットは直前のエンドゾーン（スタート位置）に戻される [cite: 162]。

## 2. モビリティ（移動）タスクの詳細とスコア
テストレーンを端から端まで走り切ることでポイントを獲得。

### 2-1. レーンの種類と仕様
* [cite_start]**Crossing Ramps（交差スロープ）:** 15度または30度の傾斜 [cite: 82][cite_start]。スリップしやすいディスクが配置される場合あり [cite: 134]。
* [cite_start]**K-Rails（K字レール）:** 10cmまたは20cmの高さのトリップ（つまずき）障害物が配置された斜面 [cite: 85, 135]。
* [cite_start]**Gravel Terrain（砂利）:** X型のフレームが埋め込まれ、足場が崩れやすい地形 [cite: 84]。
* [cite_start]**Cubic Stepfields（キュービックステップ）:** 高さ15cmのブロック状の段差が続く地形 [cite: 86]。
* [cite_start]**Traverse & Center（ドアのセンタリング通過）:** ロボットの機体幅 ＋ 10cmに設定された狭いドア（ゲート）の通過 [cite: 79-81]。
* [cite_start]**Pallets & Pipes:** 高さ20cmのパイプやパレットの段差を乗り越える [cite: 82]。
* [cite_start]**Stairs with Debris（瓦礫のある階段）:** 蹴上がり20cm [cite: 91][cite_start]。障害物（バリア）が配置され、昇り・降りのどちらかを選択可能 [cite: 90, 142]。
* [cite_start]**Doors（ドアの開閉）:** 幅90cm、レバー式ノブがついており、閉まる方向に重み（ウェイト）がかかっているドアの通過 [cite: 89]。
* [cite_start]**Avoid Holes（自律専用レーン）:** 高低差10cmのパレット状の道から落ちないように進み、幅90cmのゲートを複数通過する [cite: 88]。

### 2-2. モビリティのスコア計算
* [cite_start]**遠隔操作（Teleop）:** 1往復（片道）につき **1ポイント** [cite: 283]。
* [cite_start]**自律走行（Auto）:** インターフェースから完全に手を離した状態での片道走破につき **10ポイント** [cite: 284]。
[cite_start]※途中で操縦に介入した場合、そのラップは遠隔操作の1ポイントとなる [cite: 285]。

## 3. デクステリティ（作業）タスクの詳細とスコア
レーン内に設置されたオブジェクトをアーム等で操作する。
[cite_start]※【重要】モビリティタスクで「2ポイント」以上獲得していないと採点されない [cite: 293]。

### 3-1. 作業の種類と基本ポイント
[cite_start]対象物は壁面に対して直線的に並んだ「Linear」と、全方位に向いた「Omni」がある [cite: 211, 464]。
* [cite_start]**Inspect（点検）:** 対象物をカメラで視認（Linear: 1点 / Omni: 2点） [cite: 148, 391, 392]。
* [cite_start]**Touch（接触）:** 予選のみ。対象物の穴にツールを触れさせる（Linear: 2点 / Omni: 4点） [cite: 148, 393-395]。
* [cite_start]**Insert（挿入）:** 準決勝・決勝のみ。専用シャフトツールを対象の穴に挿入する（Linear: 3点 / Omni: 6点） [cite: 148, 396-398]。
* [cite_start]**Operational（実践操作）:** 準決勝・決勝のみ。非常停止ボタンを押す、バルブを回す、鍵を挿す（Omni配置のみ・各10点） [cite: 149, 150, 399, 400]。

### 3-2. 自律ボーナス（スコア倍率）
* [cite_start]**TA（遠隔アプローチ＋自律作業）:** ロボットの接近までは遠隔操作、作業のみ自律で行った場合 = **4倍** [cite: 298]。
* [cite_start]**AA（完全自律）:** 接近から作業完了まで完全に手を触れず自律で行った場合 = **8倍** [cite: 299]。

## 4. センシング（被災者クレート）タスクの詳細とスコア
[cite_start]※【重要】モビリティで2ポイント、デクステリティで2ポイントを獲得し、かつアームを格納状態に戻さなければ採点されない [cite: 438, 442]。

* [cite_start]**Thermal Image Acuity（熱感知） - 5点:** サーマルカメラを使用し、「C」の字パターンの熱源を2段階の深さまで識別する [cite: 444]。
* [cite_start]**Partial Image Recognition（画像/危険物認識） - 4点:** 危険物（Hazmat）ラベルを自律検知し、バウンディングボックスとラベル名を表示する [cite: 445, 446]。
* [cite_start]**Motion Detection（動体検知） - 4点:** 回転するターゲット図形を自律検知し、360度追従し続ける [cite: 447-449]。
* [cite_start]**Proximity Sampling（近接・磁気検知） - 3点:** アームの先端等に付けた磁力計を使用し、クレート内の磁石を検知する [cite: 450, 451]。
* [cite_start]**QR Code Acuity（QRコード認識） - 2点:** QRコードを自律的に読み取り、テキストを表示する [cite: 452-454]。
* [cite_start]**2-Way Audio Acuity（双方向音声通信） - 1点:** マイク/スピーカーを使用し、クレート内のMP3プレイヤーから流れる英数字の羅列を聞き取る [cite: 455-458]。
* [cite_start]**Visual/Color Acuity（視覚/色認識） - 1点:** 3段階の深さにある同心円「C」の字のギャップを識別する [cite: 459, 460]。

## 5. マッピング・探索（自律）タスクのデータ要件
[cite_start]20分間で地図を作成。競技終了後5分以内にデータを提出する [cite: 637, 655]。

### 5-1. 2Dマップ（GeoTIFF形式）
* [cite_start]必須提出だが直接のスコアにはならない（可視化の補助用） [cite: 645, 646, 663, 664]。
* [cite_start]**色指定:** 未探索はグレー市松模様、探索済みは黒グリッド、初期位置は緑矢印、壁はダークブルー、走行経路はマゼンタ [cite: 714-724, 730]。

### 5-2. 3D点群マップ（PLY形式）
* [cite_start]主催者の「正解データ（Ground Truth）」との比較（誤差と網羅率）で採点される [cite: 666, 667]。
* [cite_start]**ボーナス:** RGBカラー情報や熱（温度）情報を点群に含めると、1.25倍〜2倍のスコアボーナスが付与される [cite: 668, 669]。
* [cite_start]**スケール:** 厳密にメートル単位であること。原点(0,0,0)はロボットのスタート位置の前面中央・床の高さとし、+Y方向を正面、Z軸を垂直方向とする [cite: 796, 797, 799-801]。
* [cite_start]**データ構造例:** (ply, format ascii 1.0, property float x, y, z 等を含めること) [cite: 805-822]

### 5-3. 検知オブジェクトリスト（CSV形式）
* [cite_start]コース内のAprilTag、危険物標識、実体物、熱源などの座標をリストアップ [cite: 677, 678][cite_start]。重複報告や誤検知はペナルティ対象 [cite: 679, 683]。
* **データ構造例:**
  `detection,time,type,name,x,y,z,robot,mode`
  [cite_start]`1,14:28:01,"ar_code","2",-8.2992,-2.2904,0.49,"Robot 1",A` [cite: 750, 773]

## 6. 追加チャレンジとアワード（表彰）

### 6-1. ボーナスチャレンジ
* [cite_start]**立てかけ障害物（Negotiate Leaning Obstacles）:** マグネットで壁に固定されたバーに接触せず（落とさず）通過できれば追加ポイント [cite: 172-177]。
* [cite_start]**ペイロード運搬:** 指定の重量物をアームで把持したままレーンを走破する [cite: 180, 181]。
* [cite_start]**通信劣化テスト（Radio Comms Degradation）:** 指定の劣化ボックス（帯域5Mbps、パケットロス10%、遅延100ms）を通した過酷な通信環境下でタスクをクリアする [cite: 182-186]。

### 6-2. アワード（表彰）一覧
* [cite_start]**総合表彰:** 全能力の総合スコア上位3チーム（トロフィー） [cite: 194]。
* **Best-In-Class（証明書）:**
  * [cite_start]自律性総合 (Autonomy) [cite: 199]
  * [cite_start]自律移動 (Autonomous Mobility) [cite: 201]
  * [cite_start]自律作業 (Autonomous Dexterity) [cite: 208]
  * [cite_start]自律マッピング (Autonomous Mapping) [cite: 216]
  * [cite_start]小型ロボット (Small Robot) [cite: 218]
* [cite_start]**Bracket Classification Awards（機体別評価）:** クローラ、車輪、二足歩行、四足歩行、オープンのカテゴリごとに同種間での最高性能を表彰 [cite: 221-224]。
